# 03. ユニット・兵種仕様

## ステータス一覧

`Stats` data class で管理。

| パラメータ | 略称 | 説明 | 用途 |
|-----------|------|------|------|
| HP | HP | 体力（0で戦闘不能） | 生存判定 |
| 力 | STR | 物理攻撃力 | 物理ダメージ計算 |
| 魔力 | MAG | 魔法攻撃力 | 魔法ダメージ計算 |
| 技 | SKL | 命中率・必殺率に影響 | 命中 = SKL×2, 必殺 = SKL/2 |
| 速さ | SPD | 追撃判定・回避率に影響 | 回避 = SPD×2, 追撃差 ≥ 5 |
| 幸運 | LCK | 必殺回避・各種判定に影響 | 命中補正 = LCK/2, 必殺回避 = LCK/2 |
| 守備 | DEF | 物理ダメージ軽減 | 物理ダメージ減算 |
| 魔防 | RES | 魔法ダメージ軽減 | 魔法ダメージ減算 |
| 移動 | MOV | 移動可能マス数 | 兵種の `baseMov` から取得 |
| CT | CT | チャージタイム | SPDに基づいて蓄積、100で行動可能 |

## GameUnit クラス

### プロパティ

| プロパティ | 型 | 説明 |
|-----------|-----|------|
| `id` | String | ユニット固有ID |
| `name` | String | 表示名 |
| `unitClass` | UnitClass | 兵種 |
| `faction` | Faction | 所属陣営 |
| `level` | Int | レベル（初期 1） |
| `exp` | Int | 経験値（0〜99、100でレベルアップ） |
| `stats` | Stats | `unitClass.baseStats + personalModifier + levelUpStats` で算出 |
| `personalModifier` | Stats | 個体差補正値（Float）、デフォルト = Stats() |
| `growthRate` | GrowthRate | レベルアップ時の固定加算値（Float） |
| `weapons` | MutableList\<Weapon\> | 所持武器リスト |
| `isLord` | Boolean | 主人公ユニットかどうか |
| `currentHp` | Int | 現在HP |
| `hasActed` | Boolean | 行動済みフラグ |
| `ct` | Int | チャージタイム（初期値 0、SPDで蓄積） |
| `isDefeated` | Boolean | 戦闘不能（currentHp ≤ 0） |
| `maxHp` | Int | 最大HP（= stats.hp） |
| `mov` | Int | 移動力（= unitClass.baseMov） |

### メソッド

| メソッド | 説明 |
|---------|------|
| `equippedWeapon()` | 使用可能な先頭の武器を返す |
| `takeDamage(damage)` | ダメージ適用（0 下限クランプ） |
| `heal(amount)` | HP回復（maxHp 上限クランプ） |
| `fullHeal()` | HP全回復 |
| `gainExp(amount)` | 経験値加算。100到達でレベルアップし成長差分（StatGrowth）を返す |
| `resetAction()` | `hasActed = false` にリセット |

## ステータス計算

ユニットの最終ステータスは以下の計算式で決定される:

```
stats = unitClass.baseStats + personalModifier + levelUpStats
```

| 要素 | 説明 |
|------|------|
| `unitClass.baseStats` | 兵種の基本ステータス（兵種ごとに固定） |
| `personalModifier` | 個体差補正値（ユニット固有、デフォルト=0） |
| `levelUpStats` | レベルアップで蓄積した成長値 |

### 転職時の挙動

転職（`changeClass()`）時:
- `stats` が新クラスの `baseStats` で再計算される
- `currentHp` は新旧maxHpの差分で調整: `currentHp = (currentHp + newMaxHp - oldMaxHp).coerceIn(1, newMaxHp)`
- `personalModifier` と `levelUpStats` は保持される

## 成長システム

- 戦闘で経験値を獲得
- **100 EXP** でレベルアップ（`GameConfig.EXP_TO_LEVEL_UP`）
- レベルアップ時、各ステータスに `growthRate` の現在値（Float）をそのまま加算
- 成長は確率判定を行わない（ランダム要素なし）
- レベルアップ後、成長した HP 分だけ `currentHp` も加算される
